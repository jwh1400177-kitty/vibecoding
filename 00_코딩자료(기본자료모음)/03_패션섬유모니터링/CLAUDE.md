# 섬유패션 미디어 모니터링 시스템 — CLAUDE.md

## 프로젝트 개요

- **클라이언트:** 홍보자산개발실 (섬유패션 담당)
- **목적:** 섬유·패션 관련 기사를 매일 자동 수집 → 구글 시트 저장 → 매일 오전 8시 30분 이메일 발송
- **언어:** Python 3.10+
- **핵심 원칙:** GAS(Google Apps Script) 없이 Python 단일 스택으로 구현
- **구조:** 브랜드/경쟁사 구분 없는 키워드 기반 단일 수집 구조

## 수집 소스 (확정)

### 포털
| 소스 | 방법 | 비고 |
|------|------|------|
| 네이버 뉴스 | 검색 API (display=10, sort=date) | 출처="네이버" |
| Google RSS | feedparser (hl=ko&gl=KR) | 출처="다음" |
| 다음 뉴스 | 4단계 폴백 | 출처="다음" |

#### 다음 뉴스 4단계 폴백
1. 다음 모바일 RSS (`m.media.daum.net/m/media/newsflash/rss.xml`) → 키워드 필터
2. 다음 섹션별 RSS (사회/경제/해외) → 키워드 필터
3. Google News RSS (`site:daum.net` + 키워드)
4. Bing News RSS
실패 시 네이버만으로 진행

> ⚠️ 네이버 display=100 → 10으로 변경 (수집량 최적화, GAS 기준 적용)

### 섬유패션 전문지 17개 (사이트별 전용 RSS)

#### 전용 RSS URL 사이트 (collector/web.py `_SITE_RSS_MAP`)
| 사이트 | 전용 RSS URL |
|--------|-------------|
| TIN뉴스 | `/rss/rss_news.php?count=100` |
| 국제섬유신문 | `cdn.itnk.co.kr/rss/gn_rss_allArticle.xml` |
| 테넌트뉴스 | `/feed/?posts_per_page=50` |
| 패션저널 | `/happynews_rss.php` |
| 한국섬유신문 | `/rss/allArticle.xml` |
| 디토앤디토 | `/rss/?posts_per_page=50` |
| 피플게이트 | `/rss`, `/feed/` |

#### Google RSS 폴백 우선 사이트
어패럴뉴스, 패션인사이트, K패션뉴스, 패션리뷰, 패션비즈, 패션포스트, 미디어패션쇼, 패션지오, 텍스헤럴드, 한국섬유경제

> ℹ️ 전문지 수집 결과: v1.5 기준 13건 → v2.0 기준 ~782건 (전용 RSS 적용 효과)
> ℹ️ Google fallback은 `when:14d` 파라미터 추가

## 파이프라인 구조 (8단계)

```
1단계: 수집
       포털 (네이버 API + Google RSS + 다음 4단계 폴백)
       전문지 17개 (전용 RSS → 범용 RSS 자동탐지 → Google RSS 폴백)
       → 출처("네이버"/"다음"/"전문지"), 미디어(실제 언론사명) 태깅
       ↓
2단계: HTML 태그 정리 (clean_articles)
       ↓
3단계: 날짜 필터 (filter_by_date)
       → target_dates 범위 기사만 통과 / pub_date 빈 값 통과
       ↓
3-2단계: 블랙리스트 필터 (filter_by_blacklist)
       → 전문지: 무조건 통과
       → title에 블랙리스트 단어 포함 시 제거
       ↓
4단계: 키워드 검증 필터 (filter_by_keyword)
       → 포털(네이버/구글): 전량 통과
       → 다음: title OR summary에 키워드 있으면 통과
       → 전문지(web): title에 키워드 있으면 통과 + keyword_match 기록
       ↓
4-2단계: 화이트리스트 필터 (filter_by_whitelist)
       → 전문지: 무조건 통과
       → 섬유·패션, 산업·시장동향, 주요기업·기관명: title only (엄격)
       → 주요인물명: title + summary
       ↓
5단계: URL 중복 제거 (deduplicate_url_only)
       → 당일 배치 내에서만 중복 체크
       → keyword_category별로 분류
       ↓
5-2단계: 각 탭 3단계 중복제거 → 정렬 → 저장 (TAB_JACCARD_THRESHOLD=0.30)
       1. prefix 앞 10글자 동일 → 중복
       2. 비키워드 핵심어 1개 이상 겹침 (같은 카테고리) → 중복
       3. 2-gram Jaccard ≥ 0.30 → 중복
       → 5개 수집 탭 각각 저장
       ↓
6단계: 5개 탭 합산 → 탭 간 교차 3단계 중복제거 (JACCARD_THRESHOLD=0.30)
       → 전체종합 탭 저장
       ↓
6-2단계: 최종전송 50건 선별 (카테고리 균등 배분)
       전문지 23건 / 섬유·패션 10건 / 주요기업 8건 / 산업동향 5건 / 주요인물 4건
       소스 우선순위: 전문지 > 네이버 > 다음
       → 최종전송 탭 저장 → 이메일 발송
```

## 중복 제거 기준

### URL 중복 제거
- URL 정규화: http→https, www 제거, 트래킹 파라미터 제거
- 기사 식별자(idxno, aid, no 등) 유지
- 당일 배치 내에서만 중복 판단

### 제목 유사도 중복 제거 3단계 (탭 내부 + 전체종합)
1. **prefix 체크**: 앞 10글자 동일 → 중복 / 같은 소스 앞 8글자 동일 → 중복
2. **비키워드 핵심어 중복**: 같은 카테고리 내 공통 명사 1개 이상 → 중복
   - 제목에서 검색 키워드(ALL_KEYWORDS) 제거 후 2글자 이상 한국어 단어 추출
   - 제외 단어: 기자/뉴스/보도 등 일반 명사 24개
3. **2-gram Jaccard** ≥ 0.30 → 중복 (v2.4에서 0.45→0.30 하향)
- 소스 우선순위 높은 기사 유지, 비고에 "유사기사 N건"
- **2단계**: 탭 내부(TAB_JACCARD_THRESHOLD=0.30) → 탭 간 교차(JACCARD_THRESHOLD=0.30)

### 소스 우선순위
```
1순위: 섬유패션 전문지 (주간지/월간지/온라인)
2순위: 네이버 뉴스 (Naver API)
3순위: 다음 뉴스 (Daum)
4순위: 구글 뉴스 (Google RSS)
```

## 알려진 한계 및 주의사항

- **네이버 display=10**: GAS 기준 적용. 100→10으로 줄여도 품질 동일.
- **다음 4단계 폴백**: 1단계(모바일RSS)와 2단계(섹션RSS)는 일반 피드라 키워드 필터 후 건수가 적을 수 있음. 3단계(Google RSS site:daum.net)가 실질적 주요 소스.
- **전문지 수집 개선**: v2.0부터 전용 RSS URL 적용으로 13건→782건으로 개선.
- **패션지오/텍스헤럴드/한국섬유경제**: SSL 불안정, Google RSS 폴백 사용.
- **Jaccard 0.50**: v1.5의 0.27에서 0.50으로 상향 (GAS 기준 90%, 정상 기사 중복제거 방지).
- **URL 중복 체크**: Duplicate_Check 탭 누적 참조 금지 → 당일 배치만.
- **출처 3가지**: 네이버/다음/전문지만. Google RSS는 "다음"으로 통합.
- **미디어**: 실제 언론사명. 네이버는 API source 필드, Google RSS는 entry.source.title.
- **전체종합 탭**: 구 "이메일전송최종본" 탭. config.py SHEET_FINAL = "전체종합".
